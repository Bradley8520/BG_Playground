## Decisions Made

1. **Skills vs. agents for coaching:** Coaching behavior was implemented as a skill (always-active instructions in `copilot-instructions.md`) rather than a separate agent. Skills are the right choice when behavior should be contextual and automatic; agents are for isolated personas with their own tool scope.

2. **`docs/` folder for reference docs:** Trigger phrase reference doc was placed in `docs/` (not `skills/`) to separate reference material from executable workflow skills. Maintenance rule added to `copilot-instructions.md` to keep the ref doc updated automatically.

3. **Pandoc as backup converter:** Recommended alongside `python-docx` as a more robust alternative. `python-docx` stays as the primary method (already working, documented in skill). Pandoc is available when formatting requirements exceed what `python-docx` can deliver.

4. **`.vscode/settings.json` at workspace level:** Font and editor settings scoped to the workspace (not user-level) so they travel with the repo and don't override any global preferences outside this project.

5. **Stub skill files:** `verbatim-analysis`, `python-scripts`, and `session-start` were created as stubs rather than left uncreated. Rationale: having the file registered in `copilot-instructions.md` means Copilot will load it and find *something* to work with. Stubs grow organically as real patterns emerge.

## Learnings & Gotchas Discovered

- **Multi-line Python `-c` in PowerShell fails:** Always write Python to a `.py` file, run it, then delete. Never use `python -c "..."` for multi-line scripts in PowerShell. Documented in `skills/doc-export/SKILL.md` and `skills/python-scripts/SKILL.md`.
- **`code --install-extension` works while VS Code is open:** Extensions install to user directory immediately without needing to restart VS Code.
- **PowerShell 7 as MSIX:** `pwsh` installs as an MSIX package; accessible via WindowsApps stub. The PATH needs to be refreshed in existing terminals after install.
- **`?.Source` null-conditional not supported in PS5:** Use explicit null check. Only relevant in Windows PowerShell 5.1, not PowerShell 7.
- **Apostrophe collision in taxonomy bulk conversions:** Contractions (`didn't`, `can't`) corrupt delimiter-split operations when `'` is used as a delimiter. Use `|`, tab, or `;;` instead. Full pattern in `skills/taxonomy/SKILL.md`.
